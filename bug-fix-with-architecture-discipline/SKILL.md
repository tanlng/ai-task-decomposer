# bug-fix-with-architecture-discipline

## 何时使用
- 修这个 bug
- 这个问题怎么修
- 感觉架构不太对
- 不要打补丁
- 这个 bug 很复杂
- 同类问题反复出现
- 改一处会影响很多地方

## 工作流
1. Bug 现象重述
2. 定位计划
3. 建立观测点
4. 责任层判断
5. 修复决策
6. 架构升级判断

## 默认输出
1. Bug 理解
2. 定位计划
3. 观测点
4. 根因与归属
5. 修复方案
6. 架构建议
7. 风险
